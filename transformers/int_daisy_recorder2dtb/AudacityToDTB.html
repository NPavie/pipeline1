Overview

This script creates a DAISY/NISO NCX-Only DTB from an Audacity project. By using markers in a predefined way while recording audio in Audacity, the resulting DTB can contain headings, pagenumbers, producer notes and sidebars.

A special technique (described below) also allows the pagenumbers, producer notes and sidebars to become skippable in th eoutput DTB.

Input Requirements

An Audacity project file (*.aup), following the DTB-enabling conventions as described in the section below.

Using Audacity to create an NCX-Only DTB

For each wave track in Audacity, there must be a label track immediately below.

You can record your project using one wavetrack only, or you can use several wavetracks, as long as all wavetracks have a label track immediately below.

Creating headings

For each heading you want create, drop a Marker. The markers value must begin with h1, h2, h3, h4, h5, or h6 for the marker to be detected as a heading.

The text following in the marker value will become the heading text. For example a marker value of h2 Introduction to Marxism will create a level 2 heading with the title "Introduction to Marxism".

Creating pages

For each page you want create, drop a Marker. The markers value must begin with p for the marker to be detected as a pagenumber.

For example, the marker value "p56" will create a normal pagenumber with the value "56". You can also create special and front pages; the marker value pIVpA23 will create a page-special with the value A23.


You can also have pagenumbers be autonumbered during the DTB generation. Simply create a marker with the value p and then activate the autonumbering feature of this script. Note - if you active page renumbering, all normal pages will be renumbered, even those that were actually given a value in the Marker value field.

Creating producer notes

For each producer note you want create, drop a Marker. The markers value must begin with prodnote for the marker to be detected as a producer note.

Creating sidebars

For each producer note you want create, drop a Marker. The markers value must begin with sidebar for the marker to be detected as a sidebar.

Use markers with duration if you can!

By creating markers that have a duration (i.e. a start and an end point), you allow constructs to become skippable.

For a maximum quality output DTB, adopt this technique in the following way:

  • Create markers that spans each heading announcement

  • Create markers that spans each page number announcement

  • Create markers that spans each sidebar (the entire sidebar, not just the sidebar title announcement

  • Create markers that spans each producer note (the entire producer note, not just the producer note title announcement

Title and Author

This script will assume that the reading of the documents title will be the first phrase of the recording, and that the documents author will be the second phrase.

Metadata

Output

A DAISY/NISO (ANSI/NISO Z3986-2005) NCX-Only DTB that can contain headings, pages, producer notes and sidebars. The three latter are skippable if markers with duration are used.

Configuration

Input file
Required. The input Audacity project file (*.aup)
Output directory
Required. The directory to store the output in. This can be set as the same as the input directory, but it is recommended to use another directory than the input directory.
