Transformer Purpose

This transformer Validates an XML document against one or several schemas. Simultaneous validation using both inlined and inparam schemas is supported.

Depending on input document type and inparameters, the validation is performed using one or several validation languages; DTD validation, RelaxNG validation, XSD validation, Schematron validation, and URI resolvement checking.

The output of the transformer is not another document (although this would be a viable place to get static PSVI augmented output, if we hadnt thought of PSVI augmentation as inherently evil).

Instead, this transformer generates 0-n messages each with an associated severity level; using inparameters the user can determine which (if any) severity level should cause the transformer to cause an abort; this feature is intended to be used when this transformer is placed in a transformer sequence.

Input Requirements

Output

0-n messages, piped to the DMFC logger.

On success

0-n messages issued.

On error

0-n messages issued, and lastly a TransformerException thrown

Configuration/Customization

Parameters (tdf)

inXML
Path to the XML document to be validated
schemas
Comma separated list of schema identifiers to validate the input document against. Identifiers may be expressed as a public or system IDs. RelaxNG, W3C Schema and Schematron are allowed types of schemas. Schemas that occur inline do not need to be namedropped here, assuming that the useInlineSchemas parameter is set to true.
useInlineSchemas
Whether to perform validation using inline schemas
useFileset
Whether to perform URI resolvment checking using inline schemas
abortOn
Enumerated transformer Abort level; WARNING, ERROR, FATAL

Extended configurability

None at the moment.

Further development

Support for ISO Schematron and NVDL, all using the jaxp.validation API.
