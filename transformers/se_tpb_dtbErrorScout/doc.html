Transformer Purpose

Check feasible validity of a DTB.

This transformer is not provided to perform full conformance checking; instead primarily intended for high-speed production interim integrity checking.

ErrorScout only supports Single Volume DTBs (SVDTB).

Input Requirements

A pathspec to the manifest file of a DTB; in other words an NCC or an OPF.

Output

On success

If erroneous states are found: one or several messages with level WARNING or FATAL. If no erroneous states are found: one message level INFO, containing the string "No errors found".

On error

No specific recovery scheme. On error, this transformer will send a fatal message, then throw an exception and abort.

Configuration/Customization

Parameters (tdf)

manifest
The manifest file (ncc, opf)
scoutLevel

The scout level; values are SLIM, MEDIUM or MAXED.

Testlevel SLIM performs fundamental integrity tests on the DTB:

  • file existance
  • file readability
  • DTD Validity
  • Interdocument fragment link integrity

Testlevel MEDIUM performs all tests of level SLIM, plus:

  • execution of RelaxNG, Smil timing and Mp3 tests
  • Schematron tests on manifest files (ncc, opf)

Testlevel MAXED performs all tests of the SLIM and MEDIAN levels, plus:

  • additional (timeconsuming) Schematron tests (on smil files, etc).

Note: do not assume that level MAXED performs full conformance validation. For that, get a tool (or transformer) such as ZedVal

filesetType
The input fileset type; values are DAISY_202 or Z3986.

Extended configurability

None.

Further development

Increase the testing on 2.02 DTBs to become equivalent to full mode in the Daisy 2.02 Validator.

Increasing Z3986 DTB support is not considered useful since ZedVal exists as another transformer.
