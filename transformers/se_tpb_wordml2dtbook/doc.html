WordML2DTBook as of November 21, 2005

General assumptions

The text must be contained in a single flow. The converter has not been designed for nor tested with anchored or free floating objects.

List of addressed objects:

  • Paragraph styles
  • Character styles
  • Lists
  • Tables
  • Footnotes
  • Images

Objects not in this list are ignored, e.g. page breaks, cross references...

Paragraph and character styles

Paragraph styles

The WordML2DTBook transform will recognize the following built-in paragraph formats:

nameaction
heading 1h1
heading 2h2
heading 3h3
heading 4h4
heading 5h5
heading 6h6
footnote texta block of consecutive paragraphs with footnote text is wrapped in a note-tag and each paragraph is wrapped in a p-tag
block texta block of consecutive paragraphs with block text is wrapped in a blockquote-tag and each paragraph is wrapped in a p-tag
annotation textthe content is commented out

Character styles

The WordML2DTBook transform will recognize the following built-in character formats:

nameaction
strongstrong
emphasisem
page numberpagenum
footnote referencenoteref

Customizing

Additional styles can be defined using the custom section of the "mappings" tag set. It has been suggested that this section should be separated from the xslt.

Structure

WordML contains wrapper elements that closely matches the levelx tagging, which consequently are converted into levelx tags.

Lists

List paragraphs are converted into lists of type pl and the list item symbol is kept as contents.

Hierarchical lists will be flattened.

Tables

Tables are converted, but row spanning can't be used.

Footnotes

By using standard footnotes i Word, the link between note and reference is preserved. If this is not an option, it is possible to use the "footnote reference" character style and the "footnote text" paragraph style. However, a link between the two is not created using this method.

Images

An imagegroup and an img tag is created for an image, but the image is not extracted (this is beyond the limitations of XSLT 1.0 anyway).

Formatting

No formatting is taken into account in this transform, thus e.g. bold or italic is not converted. Bold or italics that should be retained must be tagged using the strong or emphasis character style, respectively.

Future improvements

  • Auto connect footnotes that were created using styles.
  • Handle hierarchical lists
  • Moving the customizable part out of the xslt
